# 이 폴더에서 일하는 방법 (Claude 작업 지침)

이 폴더는 **개발 연습용** 폴더입니다. 사용자는 비개발자이며, 개발을 배워가는 중입니다.
아래 규칙을 항상 지켜 주세요.

---

## 🤝 일하는 방식 (가장 중요)

1. **쉬운 말로 설명하기**
   - 개발 용어(서버, 배포, DB, 인증 등)는 비개발자도 이해할 수 있게 **풀어서** 설명합니다.
   - 어려운 단어를 쓸 땐 괄호로 쉬운 뜻을 같이 적습니다. 예) "배포(=인터넷에 올려서 누구나 볼 수 있게 하는 것)"

2. **지금 뭐 하는지 자주 말하기**
   - 작업을 시작할 때, 중간에, 끝났을 때 **"지금 ○○ 하는 중이에요"** 라고 알려 줍니다.
   - 한참 조용히 작업하지 말고, 진행 상황을 자주 공유합니다.

3. **시킨 것만, 단계적으로**
   - 사용자가 **지시한 것만** 합니다. 시키지 않은 기능을 미리 만들지 않습니다.
   - 다음 단계가 떠오르면 멋대로 진행하지 말고 **"다음은 ○○ 할까요?"** 라고 물어봅니다.

4. **할 수 있는 건 최대한 자동으로**
   - 코드 작성, 설치, 설정, 실행, 테스트 등 **컴퓨터로 할 수 있는 일은 직접** 끝까지 처리합니다.
   - **사람이 직접 해야만 하는 일**(예: 비밀번호 입력, 결제, 웹사이트 버튼 클릭 승인)만 골라서,
     **무엇을 / 어디서 / 어떻게** 하면 되는지 쉽게 안내합니다.

5. **다 만들면 바로 보여주기**
   - 개발이 끝나면 항상 **결과를 바로 확인할 수 있게 화면을 띄워(열어)** 줍니다.
   - 보통 `오이마켓`은 미리보기로 실행해서 브라우저 화면을 보여 줍니다.

---

## 🧰 이 폴더의 기술 (무엇으로 만드나)

- **Next.js** : 웹사이트(화면 + 기능)를 만드는 도구입니다. (= 웹사이트를 만드는 틀)
- **Supabase** : 회원 정보·데이터를 저장하는 인터넷 창고입니다. (= 데이터 보관소 + 로그인 기능)
- **Vercel** : 만든 웹사이트를 인터넷에 올려 주는 서비스입니다. (= 배포 = 인터넷 공개)

---

## 📁 폴더 안에 뭐가 있나

이 폴더(`ggm`) 안에는 **두 개의 프로젝트**가 들어 있습니다.

1. **가계부** — `index.html` (폴더 맨 위에 있는 파일 하나)
   - 이미 만들어져 Vercel(`ggm`)로 인터넷에 올라가 있습니다.
   - Supabase의 `transactions`(거래내역) 데이터를 사용합니다.

2. **오이마켓** — `oimarket/` 폴더 (당근마켓 같은 중고거래 사이트, 농장·농부 컨셉)
   - Next.js로 만드는 중입니다. **나중에 가계부와는 다른 주소로 따로 배포**할 예정입니다.
   - 가계부와 **같은 Supabase 창고**를 함께 씁니다.
   - 현재까지: **1단계 회원가입 / 로그인 / 로그아웃 완료**.

> 두 프로젝트가 한 폴더에 있으니, 작업할 때 **어느 쪽 이야기인지** 먼저 확인하세요.
> (특별한 말이 없으면 보통 `오이마켓` 작업입니다.)

---

## ▶️ 오이마켓 실행/확인 방법

- 미리보기 실행: 작업 폴더에서 `oimarket` 앱을 dev 모드로 띄웁니다.
  (`.claude/launch.json`에 `oimarket` 설정이 있어 미리보기 도구로 바로 실행 가능)
- 실행 후에는 **브라우저 화면(스크린샷)** 으로 결과를 사용자에게 보여 줍니다.
- 사용자가 직접 볼 때: `oimarket` 폴더에서 `npm run dev` → 브라우저에서 `http://localhost:3000`

---

## ⚠️ 개발자(Claude)가 지켜야 할 기술 메모

- **Next.js 16** 사용 → 예전 `middleware.ts`는 **`proxy.ts`**로 이름이 바뀌었습니다.
- 로그인/회원가입 등 인증은 `@supabase/ssr`로 구현되어 있습니다.
  (관련 파일: `oimarket/src/utils/supabase/`, `oimarket/src/app/auth/actions.ts`)
- Supabase 작업이 필요하면 **`supabase` 스킬**을 먼저 참고합니다. (보안·최신 문법 확인)
- 비밀 키가 들어 있는 `.env.local` 파일은 **인터넷에 올리지 않습니다**(git 제외 설정됨).
- 코드를 바꾼 뒤에는 **꼭 실행해서 잘 되는지 확인**하고, 그 결과를 사용자에게 보여 줍니다.

---

## 🙋 사람이 직접 해야 하는 일 (Claude는 대신 못 함)

이런 일은 사용자가 직접 해야 하므로, **쉽게 안내만** 합니다.

- 비밀번호·결제정보·인증번호 입력
- 웹사이트에서 "동의/승인/결제/삭제" 같은 **되돌리기 어려운 버튼** 누르기
- Vercel·Supabase 같은 외부 서비스에서 **계정 로그인 / 권한 변경 / 배포 최종 승인**
